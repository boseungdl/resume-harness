## 0. 한눈에

- **정의(한 줄)**: 고령 인구 수요와 접근성·기존 시설 공급의 불균형을 공공데이터로 계량화하고, 실제 설치 가능한 서울시 공원 중 신규 파크골프장 최적 입지 3곳을 추천한 데이터 분석 프로젝트.
- **기간**: 2025.06 ~ 2025.10 (오라클 미래내일 일경험 공공데이터 분석 과정 / 발표 2025.10.12). 문제정의·주제선정 7월 → 방법론·데이터수집 7~8월 → 모델링·기대효과 8~10월 → 발표·정책제안 10월.
- **주최/성과**: **한국오라클 「미래내일 일경험」 공공데이터 분석 프로젝트 — 우수상 수상 (2025)**. 팀명: **팀 미라클(Miracle)**.
- **인원·본인 역할**: 다전공 팀(한승보 + 허예정·이나경·김민정 + 교육과학대 스포츠응용산업학과 윤형준 등). 본인(한승보) 역할 = **문제 정의 리드 + 분석 방법론 설계 + 접근성/커버리지 분석 개발**(발표대본·형준 문서에 "커버리지 분석: 승보형의 개발"로 명시). 아래 4. 근거 참조.
- **repo**: github.com/boseungdl/park-golf — TypeScript 94% + JavaScript, `frontend/`·`scripts/`·`docker-compose.yml`, 커밋 27개(대시보드 웹 서비스). README 없음.
- **스택/기법**: Python(pandas·GeoPandas·Shapely·pyproj) · Geocoding · Min-Max 정규화 · **LASSO 회귀(변수 선택)** · **AHP(전문가 가중치)** · **베이지안 통합(LASSO=사전분포, AHP=우도)** · **MCLP(최대 커버리지 입지 최적화)** · 5km 버퍼 커버리지 분석 · 웹 대시보드(지도 시각화). *(방법론 문서에는 EFA·DBSCAN·K-Medoids·2SFCA·AutoEncoder·회귀 등 후보도 검토됐으나 최종 채택은 위 파이프라인.)*

## 1. 이력서용 소재

### 성과 bullet (동사 + 수치)
- 서울시 25개 자치구 대상 **고령 인구·교통·시설 공공데이터 10여 종을 수집·정제·공간 결합**해 자치구/행정동 단위 수요-공급 불균형 지수를 산출. *(데이터 분석가 각도)*
- **LASSO(데이터 기반)와 AHP(전문가 설문 기반) 가중치를 베이지안 방식으로 통합**해, 단일 방법 편향을 줄인 수요지수 산출식을 설계. *(애널리틱스/통계 각도)*
- **MCLP(최대 커버리지 최적화) 모델**로 30개 공원 후보 중 최적 3곳(성북·봉제산·일자산 근린공원)을 도출, 소외지역 불균형 최대 **22.8%**·3개구 평균 약 **13%** 완화를 정량 입증. *(의사결정 지원/최적화 각도)*
- **GeoPandas 5km 버퍼·행정동 면적 교차(≥50%) 커버리지 분석**으로 접근성 사각지대를 지도화하고, 입지 적용 후 왕복 이동시간 **약 26~30분 단축**(성북·강서), 강동구 거리 **2.25km** 개선을 산출. *(본인 개발 파트)*
- 공개되지 않은 **파크골프협회 회원 수(2025 최신)·파크골프장 일일 최대 수용인원**을 협회·시설에 직접 전화·메일로 확보 — 데이터 공백을 1차 수집으로 메움. *(데이터 수집 집요함 각도)*
- 분석 결과를 **서울시 공식 정책제안 홈페이지에 등록**, 공감 50개 기준 충족(부서 검토 단계) — 분석→정책 연결. *(임팩트 각도)*

### 담당 역할 상세
- **문제 정의**: 파크골프 주제 선정 근거·SMART 문제정의서 작성(승보 담당 문서 다수). 초기 "무더위 쉼터/세대통합" 주제에서 "파크골프 입지 불균형"으로 수렴시키는 과정 주도.
- **분석 방법론 설계**: 수요-공급 불균형 → 후보지 발굴 → MCLP 공간최적화 → MCDA/정밀수요예측의 4단계 파이프라인을 정리(승보 「분석 방법론」 문서). MCLP의 한계(용량 미고려 → capacitated MCLP/p-median 필요)까지 분별해 기록.
- **접근성·커버리지 분석 개발(본인)**: 공원 5km 버퍼와 행정동 경계를 GeoPandas로 교차, 동 면적 대비 50% 이상 겹침을 커버로 판정하는 스크립트 흐름. 조성 전후 사각지대·평균 이동시간 비교로 기대효과 입증.

### 기술 스택 (실무/학습 구분)
- **직접 사용(프로젝트)**: Python, pandas, GeoPandas·Shapely·pyproj(공간 연산), Geocoding, Min-Max 정규화, 좌표계 변환(EPSG:4326↔5179/5186 TM).
- **팀 협업으로 적용·설명 가능**: LASSO 회귀, AHP, 베이지안 업데이트, MCLP. *(가중치 산출·베이지안 통합은 팀원 주도 파트와 협업; 승보는 문제정의·방법론·접근성 분석 중심.)*
- **학습/검토 후 미채택**: EFA, DBSCAN, K-Medoids, 2SFCA, AutoEncoder(초기 문제정의서에 이상치 10곳 탐지용으로 구상).

## 2. 자소서용 소재

### 지원 동기·"왜" (전환점 서사)
- **오라클 데이터 교육 = 커리어 전환의 씨앗**(about-me §5-6). 승보에게 이 과정은 "기술 자체보다 **사회적 문제 해결**에 초점을 둔" 첫 경험이었고, **처음으로 '재미'를 느낀 지점**. 파크인은 그 재미의 실물 증거.
- 승보의 핵심 관점 "**사회적 문제 = 사람들 간의 문제**"와 정확히 맞물림 — 파크골프는 "고령화(건강수명 격차), 세대 갈등(공원 이용권), 디지털 격차(고령자 온라인 예약)"라는 사람 사이의 문제였고, 데이터로 그 갈등을 최소화하는 입지를 찾는 것이 목표였음.
- 발표 클로징(팀 공동 메시지, 승보 성향과 공명): *"데이터는 세상을 설명하는 좋은 도구지만, 데이터 그 자체가 세상을 의미하지는 않는다. 수치에만 매몰되지 않아야 한다."* → about-me §5 "코드는 계산기, 사람은 설계자" 관점과 일치.

### STAR 장면
- **STAR-1 (데이터 공백 돌파)**
  - S: 수요지수의 핵심 변수인 파크골프협회 회원 수가 2021년 자료까지만 공개, 파크골프장 수용인원도 상당수 비공개.
  - T: 신뢰할 수요·공급 지표를 확보해야 분석 자체가 성립.
  - A: 협회에 여러 차례 전화·메일, 시설은 한 곳씩 직접 전화해 2025 최신치·수용인원을 수기 수집.
  - R: 공개 데이터만으로는 불가능한 최신·현장 데이터 확보 → 분석 신뢰도 상승, 발표에서 차별화 포인트로 제시.
- **STAR-2 (방법론의 편향 제거)**
  - S: 수요지수 가중치를 어떻게 정할지에서 "데이터 기반 vs 전문가 감(感)"이 충돌.
  - T: 어느 한쪽만 쓰면 편향.
  - A: LASSO(데이터)로 대규모점포·경로당·지하철 수를 유의 변수로 선별하고, AHP(파크골프 카페 회원 설문)로 상대 중요도를 산출한 뒤 **베이지안으로 통합**(LASSO=사전분포, AHP=우도).
  - R: "현실성 + 통계적 타당성"을 동시에 갖춘 산출식 완성 → 심사에서 방법론 설득력.
- **STAR-3 (접근성 사각지대 시각화 — 본인 개발)**
  - S: 불균형 지수만으로는 "실제 주민이 얼마나 불편한가"가 안 보임.
  - T: 이용자 체감 효과를 눈에 보이게.
  - A: GeoPandas로 공원 5km 버퍼를 만들고 행정동 경계와 교차, 동 면적 대비 겹침 50% 이상을 커버로 판정 → 조성 전후 사각지대·평균 이동시간을 지도·수치로 비교.
  - R: 은평·강동·강서 사각지대를 드러내고, 입지 적용 후 이동시간 단축을 정량화 → 심사·정책제안의 직관적 근거.

### 트러블슈팅 서사 (분석 사고의 흔적)
- **문제**: "파크골프장이 없는 동 = 공급 부족"으로 단순 판단하면 오류(인접 시설로 충분할 수 있음).
- **가설/시도**: 그래서 공급을 2차로 나눔 — 1차 시설 수, 2차 "서비스 범위(인근 시설까지 이동거리/시간)"로 조정. 여기서 승보의 버퍼·커버리지 분석이 2차 공급을 계량.
- **분별**: MCLP는 "가까워서 편한가"만 보고 시설 용량(capacity)은 못 본다는 한계를 스스로 기록 → 용량 고려 시 capacitated MCLP/p-median 필요, "동일 규모 가정 시 MCLP로 충분"으로 범위를 정직하게 한정.
- **배움**: 모델은 만능이 아니며, 목적(불균형 해소)에 맞춰 정밀도와 단순성을 타협하는 판단이 분석의 핵심.

### 협업·리더십 에피소드
- **다전공 협업**: IT 출신(승보) + 스포츠응용산업학과(윤형준) + 데이터 팀원들. 도메인(파크골프·체육정책)은 스포츠 전공이, 데이터·모델·개발은 IT/데이터 팀원이 분담 → 승보는 "문제정의·방법론"으로 두 축을 잇는 가교 역할(about-me 7축 中 "연결하는 사람"과 일치).
- 역할 분담이 노션 Task로 문서화(문제정의서·분석정의서를 팀원별로 각자 작성 후 최종본 수렴) → 승보 성향 "모호함을 구체화"(D6)가 발현된 협업.

## 3. 포트폴리오용 소재

### 분석 파이프라인 개요 (텍스트 흐름)
1. **수집**: 고령 인구(동별), 경로당, 대규모점포, 지하철역 좌표, 버스정류소, 공원현황, 파크골프장 현황·수용인원, 협회 회원 수 등 공공데이터 + 1차 수집(협회·시설 전화).
2. **정제·공간결합**: 주소 → Geocoding → 구/동 단위 매핑(버스정류장 행정구 매핑 VWorld 포함), 좌표계 통일.
3. **정규화**: 변수별 Min-Max 정규화(비교 가능화), 수요지수는 0~1 재스케일.
4. **수요지수**: (정규화 변수 × 가중치) 합. 가중치 = LASSO → AHP → 베이지안 통합.
5. **공급지수**: 파크골프장 일일 최대 수용인원을 자치구 합산 후 Min-Max 정규화.
6. **불균형지수 = 수요지수 − 공급지수** → 상위 강서·성북·강동구 도출.
7. **후보지 필터**: 서울 전 공원 → 근린·체육공원 & 3만㎡ 이상 & 법적/환경/갈등 필터 → 30개.
8. **최적화**: MCLP(반경 5km, p=3) → 3개 조합 선정.
9. **기대효과 입증**: 불균형 해소율 재계산 + 커버리지/접근성(이동시간) 전후 비교(본인 개발).
10. **서비스화**: 웹 대시보드(불균형 지도 → 공원 마커 → 5km 버퍼 수요 합산 → MCLP 버튼 → 추천 3곳).

### 핵심 기술 의사결정 (대안 비교 → 선택)
| 결정 지점 | 대안 | 선택 | 근거 |
| --- | --- | --- | --- |
| 가중치 산출 | LASSO만 / AHP만 | **베이지안 통합** | 단일 방법 편향 제거, 데이터+전문가 동시 반영 |
| 입지 최적화 | 수요 상위 단순선택 / p-median / MCLP | **MCLP** | 특정지역 쏠림 방지, 제한된 p개로 커버 수요 최대화 |
| 커버 반경 | 1/1.5/2km, 10/15/20분 | **5km(≈20분 대중교통)** | 생활체육 평균 이동시간 선행연구 |
| 후보 공원 판정 | intersects/centroid within | **면적 겹침 ≥50%** | 동 대다수가 실제 커버돼야 유효 수요로 인정 |
| 변수축소 | EFA / 명시 변수+가중합 | 프로젝트는 **변수선택+가중합** 채택 | 수요 변수를 명확히 알기에 EFA 불필요 |
