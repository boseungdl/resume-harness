# 프로젝트 정리 — 공통 작성 지침 (서브에이전트용)

## 목적
각 프로젝트의 **날것 자료(노션·PDF·대본·코드·회의록)**를 읽고, **이력서·자소서·포트폴리오**에 바로 꺼내 쓸 수 있는 **소재 카드**로 정리한다.
최종 문장을 짓는 게 아니라, **사실·수치·장면·의사결정**을 구조화해 모아둔다.

## 지원자 (한승보) — 정체성 요약
- 취업 준비생. 웹→인프라→백엔드→데이터 전 계층을 관통. "구조를 먼저 보고 '왜'를 파는 사람."
- 희망 직무: 데이터 엔지니어 · DevOps/인프라 · 백엔드 · 데이터 분석 · PM (다직무 대응).
- 자격증: CKA · AWS SAA · 정보처리기사 · SQLD · RHCSA.
- 기준 프로필 원본: `docs/profile/about-me.md` (§9 프로젝트 상세, §3 직무별 매핑). **읽고 사실을 맞추되, 날것 자료가 원본 진실이다. 불일치 발견 시 표시.**

## 절대 규칙 (자소서 소재 품질)
1. **사실 기반**. 자료에 없는 수치·성과를 지어내지 말 것. 추정이면 "추정"이라 표기.
2. **미사여구·비유·과장 금지**. 동사·수치 중심으로 담백하게.
3. **기술 상세는 포트폴리오로, 인과·동기는 자소서로** 분리해 배치.
4. **정량 수치는 출처와 함께**. 실측 없으면 "실측치 없음"이라 명시.
5. 개인정보(전화·주민번호 등)는 옮기지 말 것.

## 산출물 형식 — `정리.md` (프로젝트 폴더 안에 저장)

```
# <프로젝트명> — 정리

## 0. 한눈에
- 정의(한 줄) / 기간 / 인원·본인 역할 / repo·링크 / 스택 / 수상

## 1. 이력서용 소재
- 성과 bullet (동사+수치, 3~6개) — 직무별 각도 표기
- 담당 역할 상세
- 기술 스택 (실무/학습 구분)
- 정량 지표 (실측/추정/미측정 명시)

## 2. 자소서용 소재
- 지원 동기·"왜" (이 프로젝트를 한 이유, 배운 의미)
- STAR 장면 (Situation-Task-Action-Result) 1~3개
- 트러블슈팅 서사 (문제→가설→시도→해결→배움)
- 협업·갈등·리더십 에피소드 (있으면)
- 성장·깨달음 한 줄

## 3. 포트폴리오용 소재
- 아키텍처 개요 (텍스트로 구성요소·흐름)
- 핵심 기술 의사결정 (대안 비교 → 선택 근거)
- 트러블슈팅 상세 (재현 가능한 수준)
- 시각 자료 목록 (원본 파일 경로 — 이미지/다이어그램/PPT)
- 코드·repo 하이라이트

## 4. 근거·출처
- 각 소재가 어느 원본 파일에서 나왔는지 (파일명)
- 불확실/검증필요 항목 목록

## 5. 직무별 활용 각도
- 이 프로젝트를 어느 직무 지원에 어떻게 전면 배치할지 (about-me §3 참조)
```

## 톤
- 담백하고 구체적으로. 표/불릿 적극 사용.
- 자료가 빈약하면 "자료 부족 — 확인 필요"로 정직하게 남길 것 (억지로 채우지 말 것).
