# 생성은 날것, 규칙은 리뷰 전용

자소서 **생성 단계(write-jaso)에는 절대 규칙·지원자 맥락을 적용하지 않는다.** 이 규칙들은 `review-jaso` 스킬에서만 적용한다.

## 동기
규칙을 생성에 넣으면 출력이 규칙에 맞춰 미리 다듬어져 "날것"을 볼 수 없다. 먼저 모델이 뽑은 날것을 보고, 필요할 때만 리뷰로 규칙을 적용해 판단·수정하는 편이 낫다고 판단했다. 또한 리뷰는 작성과 분리된 신선한 컨텍스트라야 독립 감사가 된다.

## 결정
- write-jaso ①②③는 규칙 없이 생성한다. 생성에 남는 근거 제약은 **자유도**(사실 안전장치, ADR-0003)와 형식(자수·문체)뿐이다.
- 절대 규칙·지원자 맥락·커스텀 규칙의 **유일한 적용 지점은 review-jaso**다. 규칙 원문도 review-jaso 스킬에 둔다.
- `docs/jaso/rules.md`는 제거한다 — 소비자가 review-jaso 하나뿐이라 공유 캐논 파일이 불필요하다. ADR-0004의 "규칙 분산(§5 표)"은 이 결정으로 대체된다.
- CLAUDE.md는 라우터로 남아 규칙 위치(review-jaso)를 가리킨다.

## Consequences
- 절대 규칙 1·2와 지원자 맥락(약점→강점)은 본래 *생성 전략*이라, 리뷰에서 발견되면 "다시 써라"가 되어 사실상 재생성이 된다. 담백·명확·자수 같은 필터형 규칙은 리뷰로 깔끔히 걸러진다. "날것 먼저" 방침의 의도된 트레이드오프다.
- 자유도(낮음)는 생성에 남으므로 실제 지원의 사실성은 유지된다.
