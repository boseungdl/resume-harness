# 서사=맥락이지 소설체가 아니다 · 명확 우선 · 규칙 완화(S1·S3·S6) · 그릴링 금지

> **갱신 대상**: ADR-0006(서사 규칙 S1~S5)·ADR-0008(S7). 큰 골격(생성 규칙=생성 / 폴리시=리뷰 / 2계층+카드)은 유효하되, 아래처럼 정의를 바로잡는다.

## 동기
실제 지원(씨앤토트 데이터 엔지니어)을 돌려 보니, "서사 규칙"이 **자소서를 소설체로 쓰게 만드는** 부작용이 드러났다. 사용자 진단(원문 취지):
1. **모든 문항을 관통 서사로 강제할 필요 없다** — 문항 성격에 따라 명확한 주장+근거로 끝나도 된다.
2. **S2의 진의**는 "의미 번역"이라는 추상이 아니라 — 프로필이 **꾸밈없이 일차원적으로** 적혀 있어 그대로 옮기면 공감보다 **공격(반문·의심)**을 부른다는 것.
3. **S3**은 "선언 금지"가 아니라 — **명확히 말하고 그 근거를 서사(맥락)로 풀라**는 것.
4. **S6(장면 오프닝 강제)**이 톤을 획일화했다 — 의도는 그냥 **두괄식**이었다.
5. **S7**은 유지하되 깊이가 **실제로 검증**되고 **명확**해야 한다(약점 과노출 금지).
6. **가장 근본**: 서사라는 이유로 문장을 소설처럼 쓰는 것을 서사로 **착각**했다. **문장·내용은 명확한 게 핵심이고, 서사는 맥락**이다.
7. 작성 시 매번 사용자에게 **그릴링(단계별 확인)** 하지 말고, **베스트 판단으로 바로 진행**.

## 결정
### 대원칙 (신설)
**서사는 맥락이지 문체가 아니다.** 소설체·미사여구·장면 묘사 금지. 문장과 내용은 **명확한 실무 용어로 짧게**. "서사"는 답변을 관통하는 맥락(선택의 흐름·근거)일 뿐이다.

### 생성 규칙 재정의 (write-jaso ③)
- **S1 완화** — 인생 맥락 공유는 하되 **모든 문항을 관통 서사로 만들지 않는다.** 명확한 주장+근거로 끝나는 답도 정상. 인위적 origin만 금지.
- **S2 재정의** — 프로필의 일차원적 사실을 맥락 없이 옮기면 공격받기 쉽다 → 각 사실에 무엇을 뒷받침하는지 맥락을 입힌다.
- **S3 재정의** — 강점을 **명확히 말하고** 근거(상황·행동·판단)를 붙인다. **근거 없는 선언**만 금지(선언 자체는 허용).
- **S4** — 유지(약점은 강점에 복무, 과노출 금지).
- **S5** — 유지(기술 상세는 이력서·포폴, 트레이드오프 하나를 명확히).
- **S6 재정의** — **두괄식**(결론 먼저). 장면 오프닝·telling 금지 조항 **폐기**(억지 장면이 톤을 획일화).
- **S7 보강** — 깊이 차별화 유지 + **(1)실제 검증됨 (2)명확한 표현 (3)약점 과노출 금지.**

### 진행 원칙 (신설)
매 단계 확인·그릴링 금지. **베스트 판단으로 바로 진행**하고 근거만 짧게 밝힌다. 추론 직무·자유도·각도는 스스로 결정(자유도는 명시 없으면 낮음). 정말 모호/비가역인 것만 질문.

### 리뷰 (review-jaso)
- 점검 항목에 **대원칙(서사=맥락, 소설체 아님)** 추가. S1·S3·S6은 위 재정의대로 갱신 — **서사·장면 오프닝 강제를 오히려 위반으로 적발**한다.
