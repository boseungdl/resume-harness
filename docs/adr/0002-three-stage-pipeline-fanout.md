# 3단계 파이프라인 + 문항당 병렬 팬아웃

> **⚠️ 대체됨 (Superseded by ADR-0004).** 분석→매칭→작성 구조와 매칭 검토점은 ADR-0004(리서치 → 모범답안 → 개인화)로 대체되었다. 매칭은 억지 연결을 유발해 제거했다. 아래는 이력 보존용.

자소서 생성을 분석 → 매칭 → 작성 3단계로 나누고, 오케스트레이터가 각 단계를 서브에이전트에 위임한다. 작성 단계는 **문항 하나당 에이전트 하나**로 병렬 팬아웃해 여러 문항을 동시에 작성한다. 매칭 산출물(요구역량 ↔ 프로필 근거)은 작성에 들어가기 전 사람이 검토·승인하는 지점으로 둔다 — 일반론·엉뚱한 근거를 조기에 차단하기 위함.

## Considered Options
- 2단계(분석→작성): 작성 스텝이 매칭까지 겸함 — 빠르나 통제·품질이 약해 기각.
- 단일 에이전트 순차: 서브에이전트 없이 한 흐름 — 느리고 병렬 이점 없어 기각.
