# 소스에서 해석을 벗긴다 — 사실 전용 마스터 소스와 ③a 해석 독점

> **보강 대상**: ADR-0012(③a 신설·컨텍스트 분리). ③a 컨텍스트에서 규칙을 뺐듯, 이번엔 **소스에서 해석을 뺀다.**

## 동기
③a를 신설(ADR-0012)했는데도 사용자 진단은 **"매번 같은 답이 나온다"**였다. 원인은 소스 오염:

`about-me.md`는 사실 모음으로 출발했지만 회고를 거치며 **해석·결론이 구워진 문서**가 됐다 — 핵심 정체성 7축, "한 줄 서사", "인터뷰어 종합 프레이밍", 약점 재프레이밍, 화살표(→)로 끝나는 교훈 문장, "서사의 클라이맥스" 같은 라벨. ③a에 이 문서를 주면 서사를 **새로 해석하는 게 아니라 이미 적힌 결론을 받아쓴다.** 소스에 정답이 적혀 있으면 몇 번을 돌려도 같은 서사(애매함 원점 → 겁 → 합쳐짐 → 가정)로 수렴한다. ADR-0012가 "규칙은 절제지 생성이 아니다"로 규칙-컨텍스트를 분리했다면, 이번 실패는 **소스에 박힌 해석이 생성을 고정**하는 경우다.

## 결정

### 1. `docs/profile/master-source.md` 신설 — 사실·장면·원문 전용
- 담는 것: 사건·날짜·역할·수치 등 **사실**, D1~D6 **원문 발언**, 제3자 피드백 **원문**, ⚠ 사실 경계(본인 자산 아님 / 정량 미확인 — 진실성 가드라 반드시 보존).
- 빼는 것: 해석·결론·프레이밍·직무별 강조 매핑·면접 스크립트·"한 줄 서사". **해석은 ③a가 매번 새로 한다.**

### 2. 파이프라인의 프로필 입력을 master-source.md로 교체
- ③a·③b 모두 `master-source.md`만 로드. `about-me.md`는 해석·전략 아카이브로 남기되 **파이프라인 어느 단계에도 로드 금지.**

### 3. 유지보수 규칙
- 새 사실·원문이 생기면 master-source.md에 **사실 그대로** 추가한다. 해석을 덧붙이고 싶어지면 그건 ③a 산출물(`서사.md`)이나 ADR에 쓸 내용이지 소스에 쓸 내용이 아니다 — 소스에 해석이 다시 자라나면 이 ADR의 실패가 재발한다.

## Consequences
- ③a가 같은 소스에서 매번 다른 축·각도를 찾을 자유를 회복한다 — 재실행("깊이 부족 → ③a")이 실제로 다른 재료를 낸다.
- 소스가 ~40KB → ~14KB로 줄어 ③a·③b 컨텍스트 비용도 준다.
- 기존 `서사.md`(데이터-엔지니어 마스터)는 오염된 소스로 만든 산출물 — 다음 지원부터는 ③a를 master-source.md로 재실행하는 것을 권장.
- about-me.md의 면접 스크립트·직무별 매핑은 면접 준비 등 파이프라인 밖 용도로만 쓴다.
