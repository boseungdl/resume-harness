# 리뷰는 규칙 체크에 앞서 독자 시뮬레이션과 글쓰기 감사다

> **보강 대상**: review-jaso 구조. ADR-0006(신선한 컨텍스트 감사)은 유효 — 감사의 **1차 렌즈**를 바꾼다.

## 동기
사용자 질문(2026-07-18): *"리뷰를 심리나 글쓰기 같은 부분을 참고해야 하는 건가?"* 목적함수가 이미 독자 반응(호기심·"같이 일하고 싶다", ADR-0014·0015)과 글쓰기 차별화(적은 AI, ADR-0015)로 정의된 이상, 리뷰가 규칙 준수만 검사하면 목적함수와 어긋난다 — 규칙을 다 지키고도 지루한 글이 통과한다.

## 결정
리뷰를 세 렌즈로 나누고, 각 렌즈를 **독립 서브에이전트로 팬아웃**한다(독립성 있는 단계는 서브에이전트 원칙).

1. **렌즈 A — 독자 시뮬레이션 (심리, 1차)**: 규칙을 로드하지 않은 컨텍스트가 그 회사의 채용 담당자·시니어가 되어 처음 읽듯 콜드 리딩하며 반응을 기록한다 — 첫 문단에서 계속 읽고 싶었나, 어느 문장에서 의심·지루·AI 냄새가 들었나, 다 읽고 "만나보고 싶다"가 남았나. **심리 참고는 이론 로드가 아니라 반응 시뮬레이션으로 구현한다** — 기준은 문헌이 아니라 독자의 실제 반응이다.
2. **렌즈 B — 글쓰기 감사**: 문장 단위 craft 검사 — AI가 낼 법한 문장·구조·리듬, 균질한 문장 길이, 꾸밈, 흔한 각도, 담백을 핑계로 한 밋밋함.
3. **렌즈 C — 규칙·폴리시 검증**: 대원칙·S1~S8·폴리시·경험 인용 적발(ADR-0016). 문항이 많으면 문항별 추가 팬아웃.

렌즈 A가 1차인 이유: 독자가 읽다 멈추는 글은 규칙 준수 여부가 무의미하다. A·B의 발견은 C의 위반 번호로 환원하지 말고 문장 인용으로 보고한다.

## Consequences
- `review-jaso/SKILL.md`를 3렌즈 구조로 재편. 폴리시 원문은 계속 review에만 둔다(ADR-0005).
- 수정 루프 분기(ADR-0015)와 접속: A에서 "사람이 안 느껴짐" → ③a 재실행, B에서 "흔하게 말함" → ③b 재작성.
