# 깊이는 규칙이 아니라 재료에서 — 서사·고민 재구성 단계(③a) 신설과 컨텍스트 분리

> **보강 대상**: ADR-0002(파이프라인)·0010(독자의 의자)·0011(1인칭·earning). 골격(생성/리뷰 2계층)은 유지하되 **생성 내부를 재료(③a)와 절제(③b)로 분리**한다.

## 동기
ADR-0011까지 반영(대원칙 3·earning·버전 다각화)한 산출물도 사용자 진단은 여전히 **"깊이감·고민의 흔적이 안 와닿는다"**였다. 사용자의 핵심 지적:

> "핵심은 프로필에 일차원적으로 적힌 내용이 아니라, **나란 사람의 서사를 이해하고, 그 서사를 통해 내가 그동안 해왔을 법한 — 요구되는 수준에 맞는 — 깊이 있는 고민의 흔적을 추론**하는 것."

원인 분석:
1. **규칙은 절제(filter)지 생성(generator)이 아니다.** 실패할 때마다 규칙(대원칙·S규칙)을 추가해 왔지만, 깊이 재료(그 사람이 겪었을 법한 고민)를 만들어내는 단계는 어디에도 없었다. 재료 없이 금지어 목록을 든 컨텍스트는 프로필 문장 재배열 + 안전한 표현으로 수렴한다 = median 무난 답안.
2. **컨텍스트 충돌.** write-jaso SKILL.md 하나에 오케스트레이션 + 대원칙 3개(각 장문) + S1~S7 + 카드 + 자유도 + 역할 분담이 전부 들어가, 생성 컨텍스트에 상시 ~20개의 제약이 걸렸다. 생성과 절제가 한 컨텍스트에 섞이면 생성이 진다.
3. **CLAUDE.md 라우터 오염.** 라우터에 규칙 요약이 계속 자라나 원문(스킬·ADR)과 중복·경합했다.

## 결정

### 1. 파이프라인에 ③a 서사·고민 재구성 신설 (서브에이전트, 규칙 미로드)
① 리서치 → ② 모범답안 → **③a 서사·고민 재구성** → ③b 개인화·작성 → ④ review-jaso.

③a는 프로필 전체+공고+리서치만 들고(규칙·문체 지침 없이 — 다듬으면 깊이가 죽는다):
- **서사 이해**: 인생 축 — 무엇이 움직였고 어떤 실패·전환·대가가 있었는지, 감정·이유까지.
- **고민 추론**: 그 궤적을 지나온 사람이 **공고의 요구 수준**에서 겪었을 법한 고민·갈등·판단을 문항별 후보로. 궤적이 뒷받침하면 프로필에 문자 그대로 없어도 `[추론]` 표시로 대담하게. 그 연차에 당연한 얘기는 고민이 아니다.
- 산출 `서사.md` — 매끄러운 문장 금지, 원료.

이는 ADR-0010의 "궤적 기반 추론"을 역할 분담 원칙에서 **전용 파이프라인 단계로 승격**한 것이다. S1(한 컨텍스트 통 작성)과 충돌하지 않는다 — ③a는 쓰기가 아니라 재료 생성이다(서브에이전트는 쓰기가 아니라 재료·검사에).

### 2. 규칙 원문을 rules.md로 분리 (③b에서만 로드)
- `write-jaso/SKILL.md` = 오케스트레이션만(단계·역할·자유도·진행 원칙).
- `write-jaso/rules.md` = 대원칙 1~3 + S1~S7 + 문항 카드 + 버전 생성 + 형식 (압축·중복 제거). **③b 작성 직전에만 로드** — ①②③a 컨텍스트에는 규칙이 없다.
- 폴리시 원문은 종전대로 review-jaso에만.

### 3. CLAUDE.md는 순수 라우터로 환원
규칙 요약 전부 제거 — 스킬 이름과 ADR 포인터만.

### 수정 루프 재정의
"깊이 부족"은 **재료 문제**다 → ③a 재실행(규칙 추가가 아니라). 문항 수정 → ③b만. 틀 → ②. 소스 → ①.

## Consequences
- 실패 시 규칙을 추가하던 관성을 끊는다 — 규칙 추가는 절제 문제일 때만, 깊이 문제는 ③a로.
- 생성 컨텍스트가 가벼워진다: ③a는 제약 0개(자유도·궤적 근거만), ③b는 rules.md 1회 로드.
- `서사.md`가 지원 건별 산출물로 남아 수정 루프·면접 준비 재료로 재사용된다.
- CLAUDE.md·SKILL.md·rules.md·review-jaso 4곳의 중복이 제거돼 각 규칙의 원문이 정확히 한 곳에 있다.
