# 서사 규칙은 생성에서 · 자소서는 서사, 기술 상세는 이력서·포폴

자소서 생성(write-jaso ③)에 **서사 규칙(S1~S5)**을 적용한다. 리뷰(review-jaso)에는 서사 규칙 **준수 검증** + 담백·명확·자수 같은 **폴리시** + 커스텀만 남긴다.

## 동기
ADR-0005의 "생성=규칙 0(날것)"을 실제로 돌려 보니(신입~경력 2년 백엔드 시험), 규칙 없는 날것이 **프로필 항목을 문장으로만 바꿔 슬롯에 끼우는** 결과를 냈다("N+1을 resultMap JOIN 1회로 축소" 같은 §8 재서술). 사용자가 좋은 예시를 제시해 진단이 명확해졌다: 좋은 자소서는 같은 사실을 **하나의 인과 서사로 '의미 번역'**한 것이다. 이런 서사 구성은 사후 리뷰로 "넣을" 수 없다 — 없으면 전체를 다시 써야 한다. 따라서 생성 전략이다.

## 결정: 서사 규칙 (write-jaso ③)
- **S1. 관통하는 인과 서사** — "왜 이 길·무엇을 원하나"를 축으로 각 단계가 다음의 동기가 되게. 독립 일화 나열 금지.
- **S2. 사실 → 의미 번역** — 프로필 항목을 그대로 문장화하지 않고 서사에서 무엇을 증명하는지로. ("프로필을 그대로 가져온다"의 직접 해법)
- **S3. 강점은 '상황이 요구했다'로** — 형용사 선언 대신 필연으로 드러냄.
- **S4. 편력·약점은 여정으로 재구성하되 약점 과노출 금지** — 정직하되 강점 서사에 복무. **정직함이 목적이 아니라 강점 서사가 목적.**
- **S5. 자소서는 서사, 기술 상세는 이력서·포폴** — 스택·수치·문제해결 과정을 자소서에서 깊이 풀지 않고 서사 앵커로만 최소 사용. 자소서가 증명할 것은 '어떤 사람이고 왜 이 길인가'.

## 결정: 리뷰 (review-jaso)
- 서사 규칙 S1~S5 **준수 검증** + 폴리시(담백·명확·자수) + 자유도 + 커스텀. 규칙 원문은 review-jaso에 둔다.
- ②(모범답안)도 기술 나열이 아니라 **서사 구조** 중심 틀을 만든다(S5).

## Consequences
- ADR-0005의 "절대 규칙·맥락 전부 리뷰 전용"을 대체. 순수 "규칙 0" 날것은 포기한다.
- 담백·명확·자수는 여전히 리뷰의 필터형 규칙(사후 교정 가능).
- 자유도(낮음)는 생성에 유지되어 사실성 보장. 단 S2·S4의 재구성이 사실을 왜곡하지 않도록 리뷰의 자유도 검증(항목 8)이 안전망.
