# 리서치 → 모범답안 → 개인화 파이프라인 (매칭 제거, 규칙 분산)

> **갱신 (ADR-0005).** 아래 "규칙 분산(rules.md §5 표로 각 단계에 규칙 배분)" 부분은 폐기됐다. 규칙은 생성 단계에 아예 넣지 않고 `review-jaso`에서만 적용한다(생성=날것). `docs/jaso/rules.md`도 제거됐다. 파이프라인 구조(리서치→모범답안→개인화, 매칭 제거, ③은 메인)는 유효.

자소서 파이프라인을 **분석→매칭→작성**(ADR-0002)에서 **리서치 → 모범답안 → 개인화**로 재편한다. 동기: 메인 에이전트가 모든 규칙을 한 컨텍스트에 들고 있으니 규칙이 서로 희석돼 "따로 논다".

## 결정
- **① 리서치 (서브에이전트, 웹)**: 공고/직무를 웹으로 조사해 기업·직무·합격 자소서 패턴·JD를 소스로 확보. 규칙·프로필을 로드하지 않는다.
- **② 모범답안 (서브에이전트)**: 소스만으로 **프로필 무관 자유 모범답안**을 생성. ③에서 갈아끼울 틀(구조·논리·톤·설득 각도). 문체 규칙(설득·담백·명확)만 적용.
- **③ 개인화·작성 (메인 에이전트 직접)**: 모범답안 틀 위에 프로필 실제 경험을 얹는다. 절대 규칙 전부 + 지원자 맥락 + 자유도 적용. 메인이 하는 이유: 프로필이 커서 한 번만 로드하려면 메인이 적합하고, 최종 작성 판단도 메인에 둔다.
- **규칙 분산**: `docs/jaso/rules.md` §5 표대로 각 단계에 필요한 규칙만 준다. 이것이 "규칙이 따로 논다"의 근본 해법이다.
- **검증 분리**: 완성본은 `review-jaso` 스킬이 신선한 컨텍스트에서 독립 감사(2차 방어선).

## 매칭 제거
요구역량 ↔ 프로필 근거를 표로 확정하는 **매칭**(ADR-0002)을 제거한다. 요구사항마다 경험을 억지로 끼워 맞추면 연결이 어색·억지스러워지고, 절대 규칙 1(표면 매칭 금지)에 어긋난다. 대신 모범답안 틀 위에 서사를 자연스럽게 얹는다.

## Consequences
- ADR-0002(3단계 + 매칭 검토점)를 대체한다.
- CLAUDE.md는 라우터로 축소하고, 규칙 원본은 `docs/jaso/rules.md`로 이전한다 — 규칙은 항상 로드되지 않고 스킬 실행 시 필요한 부분만 로드된다. 따라서 자소서 작업은 반드시 write-jaso/review-jaso로 라우팅해야 한다.
- ② 모범답안은 창작 자유라 사실과 다를 수 있으나, ③에서 프로필 기반으로 갈아끼우므로 최종본의 사실성은 자유도(낮음)로 보장된다.
