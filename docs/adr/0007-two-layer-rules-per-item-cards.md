# 규칙 2계층 정리 + 문항별 카드 (개수 상한·중복 제거·서브에이전트는 검사)

자소서 규칙을 **2계층 + 문항별 카드**로 정리한다. 사용자가 제안한 대규모 규칙 목록(공통 style ~12 + 문항별 ~6×5 ≈ 40)을 냉정히 검토해, 내용은 살리되 구조 문제를 고친 결과다.

## 검토에서 드러난 문제
- **양이 임계치 초과** → 한 문항에 18개 금지를 동시에 들면 희석("규칙이 따로 논다"가 양으로 재현).
- **헤드라인 충돌** → "두괄식 짧게(결론 먼저)" vs "장면에서 시작(telling 금지)"는 양립 불가.
- **3계층 중복** → S1~S5 / 공통 / 문항별이 같은 말을 반복(기술 상세 빼기, 성품 주장 금지, 구체 장면 등).
- **문항별 서브에이전트가 S1 붕괴** → 문항을 나눠 쓰면 공유 인생 축이 사라져 문항이 따로 논다.
- **폴리시(금지) 비중 과다** → 표면만 깎고 설득 엔진(S1·S4)은 못 만든다.

## 결정
1. **생성 서사 규칙 S1~S6** (write-jaso ③, 메인, 긍정형·최대 6): S1 관통 인과 서사, S2 사실→의미 번역, S3 강점·성품은 상황·행동으로, S4 편력→여정(약점 과노출 금지), S5 서사·기술은 포폴(기법 나열 대신 트레이드오프 하나), S6 장면으로 열고 체현(순수 두괄식 폐기 — 충돌 해소).
2. **폴리시** (review-jaso, 금지형, 원문 한 곳): AI 문어체·거창한 비유·스택 name-drop·당연지식 성과화·타사 비교·명확/짧게. write-jaso엔 원문을 두지 않는다(중복 제거).
3. **문항별 카드** (write-jaso, 목적 1줄 + 금지 1~2): 지원동기(동기는 동기다 / 억지 origin·부정형 오프닝·기업정보 나열 금지) 등. 공통과 겹치는 항목은 카드에서 뺀다.
4. **서브에이전트는 쓰기가 아니라 검사에.** ③ 작성은 메인이 통으로 해 인생 축(S1)을 보존하고, 문항별 병렬 점검이 필요하면 review 단계에서 팬아웃한다.

## Consequences
- ADR-0006의 S1~S5를 S1~S6으로 확장·정련(0006은 유효, 이 ADR이 세부를 갱신).
- 두괄식 규칙 폐기 → "결론은 먼저 서되 장면으로 체현"으로 통일.
- 문항별 ②~⑤ 카드는 초안이며 사용자 지침으로 계속 다듬는다.
